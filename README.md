# ✈️ Travel Journal — React Project

This project is part of my initial journey into the **React** ecosystem, completed as part of the [Scrimba](https://scrimba.com/learn-react-c0e?via=u01afpl) React Course.

## 📝 Project Overview

The **Travel Journal** serves as a perfect bridge between **Vanilla JavaScript** and **React**. While it doesn't showcase the full power of the library yet (like state or effects), it effectively demonstrates the core concepts of component-based architecture and dynamic data handling.

Building this project was an engaging experience that provided a very smooth and enjoyable introduction to modern frontend development.

## 🚀 Key Features & Learning Outcomes

* **Component-Based Architecture:** Breaking down the UI into reusable functional components.
* **Data Driven UI:** Using a separate `data.js` file to manage content.
* **Props:** Passing data dynamically from a parent component to child components.
* **Mapping Data:** Implementing the `.map()` method to render a list of travel destinations efficiently.
* **Semantic HTML & CSS:** Maintaining a clean, responsive design that stays true to the original mockup.

## 🛠️ Tech Stack

* **React** (Functional Components)
* **JavaScript** (ES6+)
* **CSS3** (Flexbox, custom styling)
